## ✅ 必須の設定（執筆開始前）

### 2. 最初のタスクを決定

- [ ] **オプションA**: プロローグから順番に執筆
- [ ] **オプションB**: 自分が書きたいセクションから開始
- [ ] **オプションC**: AIに提案してもらう（推奨）

決定したら `progress.md` の「次のタスク」セクションを更新

### 3. バージョン管理の設定（推奨）

```bash

# .gitignoreの作成（必要に応じて）
echo ".claude/" > .gitignore

# 初回コミット

# （オプション）リモートリポジトリに接続
# git remote add origin <your-repository-url>
# git push -u origin main
```

- [ ] Gitリポジトリを初期化した

## ❓ よくある質問

### Q: どのくらいの期間で完成する？

A: 書籍の規模と執筆ペースによります。目安：
- 1日1セクション（2-4時間）→ 約3ヶ月
- 週3セクション → 約6ヶ月
- 週1セクション → 約1年半

ただし、品質を優先し、無理のないペースで進めることを推奨します。

### Q: 途中で方針を変更してもいい？

A: もちろんです！このチェックリストの項目は後から変更可能です。
- スタイルガイドは随時更新
- テンプレートは追加・修正可
- ワークフローも調整可

### Q: AIの提案を採用しないといけない？

A: いいえ。AIは提案するだけで、最終判断は常に人間が行います。
直感や好みを優先して構いません。

### Q: 行き詰まったら？

A: 以下を試してください：
1. 別のセクションに取り組む
2. 既存のセクションを読み返す
3. 休憩を取る
4. AIに `plan-next.md` で新しい視点を求める

## 📊 進捗の可視化（オプション）

モチベーション維持のため、進捗を可視化する方法：

1. **progress.mdで確認**
   - 完成したセクション数を数える
   - パーセンテージを計算

2. **Gitのコミット数**
   ```bash
   git log --oneline | wc -l
   ```

3. **文字数カウント**
   ```bash
   find chapters/ -name "*.md" -exec wc -w {} + | tail -1
   ```

4. **SNSでシェア**（オプション）
   - 進捗をツイート
