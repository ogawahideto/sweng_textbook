# Session Log Update

セッションログを更新します。

## 重要：追記ルール

**同じ日に複数回更新する場合は、既存の内容を上書きせず追記する。**

- 既存のセッション内容は保持
- 新しいセッションは `## セッションN（時間帯）` として追加
- 概要は一日の終わりにまとめて更新可

## 手順

1. 最新のセッションログファイルを確認する（`.ai/sessions/`）
2. 前回のログ日付と今日の日付を比較
3. 未記録の日がある場合、gitログから活動内容を抽出
4. 新しいセッションログを作成、または**既存ファイルに追記**

## 実行内容

### Step 1: 既存ログの確認

```bash
ls -la .ai/sessions/
```

### Step 2: 今日の日付を取得

今日: $CURRENT_DATE

### Step 3: Gitログから活動を抽出

```bash
git log --oneline --since="yesterday" --until="now"
```

### Step 4: セッションログの作成または追記

以下のフォーマットで `.ai/sessions/YYYY-MM-DD.md` を作成または追記:

#### 新規作成の場合

```markdown
# セッションログ - YYYY-MM-DD

## 概要
[1-2行で今日の活動を要約（一日の終わりに更新可）]

## セッション1（午前 / 午後 / 夜など）

### 主な活動
- [活動1]
- [活動2]

### 作成・更新したファイル
- [ファイル1]
- [ファイル2]

### 次のステップ
- [次にやること]
```

#### 同じ日に追記する場合

既存の内容の後に新しいセッションを追加:

```markdown
## セッション2（午後）

### 主な活動
- [活動1]
- [活動2]

### 作成・更新したファイル
- [ファイル1]

### 次のステップ
- [次にやること]
```

#### メトリクス（オプション、一日の終わりに追加）

```markdown
## メトリクス
- 稼働時間: [HH:MM]
- トークン数: [Total Tokens] (In: [Input] / Out: [Output])
- AIモデル: [Model Name]
- Gitコミット数: [N]
```

### Step 5: 結果を報告

作成または更新したログファイルの内容を表示し、コミットする。
