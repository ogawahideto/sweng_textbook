# Positive Expression Check

指定されたファイルまたはディレクトリ内のMarkdownファイルをスキャンし、ネガティブな表現を検出してポジティブな代替案を提案します。

## 使い方

```
/positive-check [ファイルパス または ディレクトリパス]
```

引数がない場合は `chapters/` ディレクトリ全体をスキャン。

## 検出対象のネガティブ表現

| パターン | 推奨する代替表現 |
|---------|----------------|
| 危険 | 可能性が眠っています / チャンスがあります |
| 失敗 | 成功に近づく / うまくいく |
| 難しい / 困難 | 奥深い / 挑戦しがいがある / 工夫が活きる |
| 問題 / 課題（ネガティブ文脈） | 機会 / 伸びしろ / テーマ |
| 罠 / 落とし穴 | コツ / ポイント / 意識すべき点 |
| 悪い例 / アンチパターン | シンプルな例 / 基本的なアプローチ / さらに効果的な方法 |
| 欠点 / デメリット | 特徴 / トレードオフ / 考慮点 |
| 間違い / 誤り | 改善の余地 / 発展の可能性 |
| 衝突 / 矛盾 | バランス / 両立 / 調和 |
| 不満 / 困っている | 期待 / 実現したいこと |
| 見落とし / 盲点 | さらに広げられる視点 / 追加できる観点 |
| 〜できない | 〜するとさらに良くなる |
| 〜を避ける | 〜を心がける |
| 妨げる / 阻害 | 〜に注意を払う |
| 〜してはいけない | 〜すると効果的 |

## 実行手順

### Step 1: 対象ファイルの特定

引数で指定されたパス、またはデフォルトの `chapters/` ディレクトリ内の `.md` ファイルをリストアップ。

### Step 2: ネガティブ表現の検索

各ファイルに対して、上記パターンを検索:

```bash
grep -n -E "危険|失敗|難しい|困難|問題|課題|罠|落とし穴|悪い例|アンチパターン|欠点|デメリット|間違い|誤り|衝突|矛盾|不満|困って|見落とし|盲点|できない|避ける|妨げ|阻害|してはいけない" [ファイル]
```

### Step 3: 文脈を確認

検出された箇所の前後を読み、本当にネガティブな文脈かを確認。
（例: ユーザーの発言の引用、ペルソナの悩み描写は許容）

### Step 4: 結果をレポート

```markdown
## ネガティブ表現チェック結果

### [ファイル名]

| 行 | 検出表現 | 文脈 | 推奨代替 |
|----|---------|------|---------|
| 15 | 「危険です」 | 解説文 | 「可能性が眠っています」 |
| 42 | 「落とし穴」 | 見出し | 「コツ」「ポイント」 |

### サマリー

- スキャンファイル数: X
- 検出箇所: Y
- 要修正: Z（ユーザー発言等を除く）
```

### Step 5: 修正の提案

検出された箇所について、具体的な修正案を提示。ユーザーが承認すれば修正を適用。

## 注意事項

- ユーザーの発言やペルソナの悩み描写はネガティブ表現を許容
- 技術用語（例: 「問題解決」）は文脈で判断
- `style-guide.md` の詳細ルールを参照
