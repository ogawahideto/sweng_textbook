# /book-polish スキル

書籍的可読性の7ルール（style-guide.md §書籍的可読性のルール）に基づいて、指定したMarkdownファイルまたはディレクトリをスキャンし、違反をレポートします。セクション末尾の順序（ルール⑥）は自動修正します。他のルールは違反箇所と修正案を提示します。

## 使い方

```
/book-polish [パスまたはディレクトリ]
```

例:
- `/book-polish chapters/part1/chapter01/1-2.md` — 1ファイルをチェック
- `/book-polish chapters/part1/chapter02/` — ディレクトリ内の全.mdをチェック

## スキャン対象ルール

| ルール | チェック内容 | 対応 |
|--------|------------|------|
| ① 見出し密度 | H2セクション内のH3が5個超 | 警告+格下げ候補の提示 |
| ② 太字の過多 | 1,000字あたり `**` が15個超 | 警告+削減候補の提示 |
| ③ AI詠唱例の配置 | セクション末尾以外での `## AIへの詠唱例` の有無（将来対応） | 情報のみ |
| ⑤ 図の前後文 | `![` の直前行が空行のみ | 警告+誘導文の追加提案 |
| ⑥ セクション末尾の順序 | `さらに学ぶ` が `AI詠唱例` より前 | **自動修正** |
| ⑦ コラムの上限 | `> ` が15行以上連続 | 警告+分割/移動の提案 |
| ⑧ 前節参照 | `^前節で` `^ここまでで` `^本節では` などのパターン | 警告+書き換え案の提示 |

## 実行手順

### Step 1: ファイルを特定する

引数がディレクトリの場合は `**/*.md` で全Markdownファイルを列挙する。ファイルが指定された場合はそのファイルのみ処理する。

### Step 2: 各ファイルをルール別にスキャン

#### ルール① 見出し密度チェック

H2見出しで区切られたセクションごとに、H3の数を数える。5個を超えるセクションを報告する。

```python
# 概念コード（実際にはファイルを読み込んで解析する）
sections = split_by_h2(content)
for section in sections:
    h3_count = count_h3(section)
    if h3_count > 5:
        warn(f"H3が{h3_count}個（推奨:5個以下）")
        suggest_h3_to_bold(section)
```

#### ルール② 太字の過多チェック

ファイル全体の文字数と `**` の出現回数を計算する。1,000字あたり15個超で警告。

#### ルール⑤ 図の前後文チェック

`![` で始まる行の直前行が空行や `---` のみの場合、誘導文が不足している可能性がある。また `![` の直後行が空行または次の見出し・リストの場合、読み解き段落が不足している可能性がある。

#### ルール⑥ セクション末尾の順序チェックと自動修正

各ファイルで `## さらに学ぶためのリソース` が `## AIへの詠唱例` より前にある場合を検出し、自動的に順序を入れ替える。

具体的には：
1. `## AIへの詠唱例` ブロック（次のH2まで）を抽出
2. `## さらに学ぶためのリソース` ブロック（次のH2まで）を抽出
3. `さらに学ぶ` ブロックを `AI詠唱例` ブロックの後ろに移動

#### ルール⑦ コラムの上限チェック

`>` で始まる行が15行以上連続する箇所を検出し、場所と行数を報告する。

#### ルール⑧ 前節参照チェック

以下の正規表現にマッチする行を報告する：
- `^前節で[はにもが]`
- `^ここまで[でに]`
- `^本節では`
- `^この節では`

### Step 3: レポートを出力する

各ファイルについて違反の有無をまとめて表示する。

```
=== book-polish レポート ===

📄 chapters/part1/chapter02/2-2.md
  ① 見出し密度: 「## クラス図の描き方」内にH3が7個（推奨: 5個以下）
     格下げ候補: ### コツ①、### コツ②、### コツ③
  ② 太字の過多: 1,000字あたり約23個（推奨: 15個以下）
  ⑤ 図の前後文: 3箇所で誘導文なし（行番号: 45, 78, 120）
  ⑥ セクション末尾の順序: 自動修正しました ✅

📄 chapters/part1/chapter02/2-3.md
  違反なし ✅

合計: 2ファイルをスキャン、1ファイルに違反あり、1ファイルを自動修正
```

### Step 4: 修正の実施

- ⑥の自動修正はレポートと同時に適用する（バックアップ不要、gitで管理）
- 他のルールは修正案を提示し、ユーザーの確認を取ってから適用する

## 注意事項

- 自動修正（⑥）はgitで追跡されているため、不要であれば `git checkout -- <ファイル>` で元に戻せます
- ルール①②の「目安」は機械的な数値です。文脈によっては適切な場合もあるため、レポートを参考に人間が最終判断してください
- このスキルはスキャン専用です。本格的な書き直しが必要な場合は `/dense-coupling` を使用してください
