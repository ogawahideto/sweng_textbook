# Estimate Pages

原稿ファイルの分量から書籍としてのページ数を概算します。

## 使い方

```
/estimate-pages              # 全体のページ数を計算
/estimate-pages chapter01    # 特定の章のみ
/estimate-pages --detailed   # セクション単位の詳細表示
/estimate-pages --format b5  # B5判で計算
```

## 計算基準

### 判型別の文字数/ページ

| 判型 | 文字/ページ |
|------|------------|
| A5（デフォルト） | 800 |
| B5 | 1200 |
| A4 | 1500 |

### 換算ルール

| 項目 | 換算値 |
|------|--------|
| 本文 | そのまま文字数 |
| コードブロック | 1行 = 50文字 |
| 図・表 | 1点 = 400文字（0.5ページ） |
| 補正係数 | ×1.1（余白・章扉等） |

## 実行手順

### Step 1: 原稿ファイルを収集

`chapters/` 以下の `.md` ファイルを再帰的に収集。

除外するファイル:
- `*-proposal.md`（企画書）
- `README.md`

### Step 2: 各ファイルを解析

各Markdownファイルに対して以下をカウント:

1. **本文文字数**: マークダウン記法（`#`, `**`, `*`, リンク記法等）を除いた純粋なテキスト
2. **コードブロック行数**: ` ``` ` で囲まれた部分の行数
3. **図・表の数**: `![` で始まる行（画像）、`|` を含むテーブル行

### Step 3: ページ数を計算

```
総文字数 = 本文文字数 + (コード行数 × 50) + (図表数 × 400)
ページ数 = ceil(総文字数 ÷ 文字数/ページ × 1.1)
```

### Step 4: 結果を表示

## 出力形式

### 標準出力

```markdown
## 📖 ページ数概算（A5判）

### 全体サマリー

| 項目 | 値 |
|------|-----|
| 総文字数（本文） | 123,456 文字 |
| コードブロック | 1,234 行 |
| 図・表 | 45 点 |
| **概算ページ数** | **約 180 ページ** |

### 章別内訳

| 章 | 本文 | コード | 図表 | ページ |
|----|------|--------|------|--------|
| プロローグ | 3,200 | 0 | 2 | 5 |
| 第1章 | 28,500 | 450 | 12 | 42 |
| 第2章 | 31,200 | 620 | 15 | 48 |
| **合計** | **123,456** | **1,234** | **45** | **約 180** |

---
📝 この値は目安です。実際のページ数は組版・レイアウトにより変動します。
```

### 詳細出力（--detailed）

章ごとにセクション単位の内訳を表示:

```markdown
### 第1章: ドメインという名の異世界探検

| セクション | 本文 | コード | 図表 | ページ |
|-----------|------|--------|------|--------|
| 1.1 なぜ聴くのか？ | 5,200 | 80 | 3 | 8 |
| 1.2 どう聴くか？ | 6,100 | 120 | 4 | 10 |
| **章計** | **28,500** | **450** | **12** | **42** |
```

## 参考: 一般的な技術書のページ数

| 分類 | ページ数目安 |
|------|------------|
| 入門書・薄い本 | 100〜150 |
| 標準的な技術書 | 200〜350 |
| 網羅的な解説書 | 400〜600 |
