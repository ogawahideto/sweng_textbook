# Workflow: Plan Next Task

次に取り組むべきタスクを決定するワークフローです。

## Trigger

- セクションが完成した後
- 執筆計画を立てたい時
- 行き詰まりを感じた時
- 定期的な進捗確認時

## Prerequisites

- [ ] `progress.md` が最新の状態に更新されている

## Steps

### 1. Load Current State

現在の状況を把握：

```
@progress.md の全内容
@plan.md の書籍構成
最近完了したタスク
```

### 2. Invoke Agent for Planning

以下のプロンプトでタスク提案を依頼：

```
あなたは『「楽しい」ソフトウェア工学の教科書 - デジタル・アルケミー』のプロジェクトマネージャーです。

@agents.md の "When Planning Tasks" ガイドラインに従ってください。

**現在の状況**:
```
@progress.md の内容
```

**最近完了したタスク**:
- [タスク1]
- [タスク2]

**考慮事項**:
1. 論理的な流れ（章の順序、依存関係）
2. バランス（各部の進捗、理論と実践）
3. 執筆者の状況（今、何を書きたいか）

**期待する形式**:
@agents.md の "Response Format > For Task Planning" に従ってください。

次に取り組むべきタスクを提案してください。
```

### 3. Agent Response Format

エージェントは以下の形式で応答：

```markdown
## Current Progress Analysis

**Overall**: 全86セクション中、12セクション完成（14%）

**By Part**:
- プロローグ: 0/1 (0%)
- 第1部: 3/6 (50%)
- 第2部: 6/6 (100%) ✓
- 第3部: 2/6 (33%)
- 第4部: 0/5 (0%)
- 付録: 1/3 (33%)

**Recent Momentum**: 第2部に集中、高い完成度

## Recommended Next Task

**Task**: 第3部 5.1節「テスト駆動開発（TDD）のリズム感」の執筆

**Reason**:
1. 第2部（実装）が完成したので、自然な流れで第3部（テスト）へ
2. TDDは第2部の実装手法と密接に関連
3. 重要度が高く、書籍の核となるセクション
4. 最近の執筆リズムを維持できる

**Expected Output**:
- 5.1節の草稿（2000-4000文字）
- TDDの実践的なコード例
- AI時代のTDD手法の解説

**Estimated Effort**: 中（2-3時間の執筆）

## Alternative Options

### Alternative 1: プロローグの執筆

**Task**: プロローグ全体の執筆

**Reason**:
- 書籍の方向性を明確化
- 第2部完成により、書くべき内容が見えてきた
- モチベーション向上（読者への最初のメッセージ）

**Pros**: 書籍全体の雰囲気を確立できる
**Cons**: 全体が見えてから書く方が良い可能性

### Alternative 2: 第1部の残りを完成

**Task**: 1.2節、2.3節の執筆

**Reason**:
- 第1部を完成させ、達成感を得る
- 基礎理論部分の充実

**Pros**: 第1部完成で大きなマイルストーン
**Cons**: 理論が続き、執筆疲れの可能性

### Alternative 3: 付録の充実

**Task**: 「AIへの詠唱集」の作成

**Reason**:
- 気分転換になる
- 既存セクションから抽出できる
- 実用性が高く、モチベーション維持

**Pros**: 軽い作業、すぐに形になる
**Cons**: 本文が優先されるべき

## Recommended Roadmap (Next 5 Tasks)

1. **5.1節 TDDのリズム感** [推奨]
   - 第2部からの自然な流れ
   - 重要度: 高

2. **5.2節 AIによるテストケース生成**
   - 5.1の続き、勢いに乗る
   - 重要度: 高

3. **プロローグ執筆**
   - テスト章で一区切り後
   - 書籍の顔を整える

4. **第1部 残りセクション完成**
   - 第1部を完結させる
   - 基礎理論を固める

5. **第3部 残りセクション**
   - 第3部全体を完成
   - 重要度: 中-高

## Strategic Considerations

### Momentum

現在、良い執筆リズム。これを維持するには：
- 難易度が適切なタスクを選ぶ
- 興味が高いトピックから着手
- 完成の達成感を重視

### Balance

第2部が完成したので：
- 他の部もバランス良く進める
- ただし、流れを重視（第3部へ）

### Quality vs Speed

- 完成度を保ちつつ、進捗も重視
- 80%の完成度で先に進み、後で磨く戦略も有効

## Questions for Author

決定に役立つ質問：

1. **今、何について書きたいですか？**
   - TDD/テスト
   - 基礎理論
   - プロローグ
   - その他

2. **執筆のペースは？**
   - 勢いに乗って連続執筆
   - ゆっくり丁寧に
   - 気分で選びたい

3. **優先事項は？**
   - 論理的な順序
   - モチベーション
   - 重要セクションから

## Recommendation Summary

**Primary**: 5.1節「TDDのリズム感」
**Fallback**: プロローグ または 第1部完成

**Next Step**:
著者の意向を確認し、タスクを決定後、`draft.md` ワークフローへ
```

### 4. Human Decision

エージェントの提案を参考に、自分で決定：

**考慮すべき点**:
1. 今の気分・モチベーション
2. 利用可能な時間
3. 知識・経験（そのトピックへの理解度）
4. 全体のバランス

**決定方法**:
- 推奨タスクを採用
- 代替案を選択
- 完全に独自のタスクを選ぶ

### 5. Update Plan

決定したタスクを `progress.md` に記録：

```markdown
## 次のタスク

1. [x] 5.1節「TDDのリズム感」の執筆
   - 予定: 2026-01-18
   - 目標: 草稿完成

2. [ ] 5.2節「AIによるテストケース生成」の執筆
   - 予定: 5.1完成後

...
```

### 6. Start Execution

次のワークフローへ：

```
選択したタスクが新規執筆 → @workflows/draft.md
選択したタスクが修正 → @workflows/review.md
```

## Output

- 明確な次のタスク
- 更新された `progress.md`
- （オプション）中期的なロードマップ

## Decision Patterns

### Pattern 1: 論理的順序優先

```
章の順序に従って、着実に進める
適用: 体系的に学べる教科書を目指す場合
```

### Pattern 2: モチベーション優先

```
今、書きたいセクションから着手
適用: 執筆の楽しさを重視、行き詰まり回避
```

### Pattern 3: 重要度優先

```
書籍の核となるセクションから完成
適用: 質を最優先、時間制約がある場合
```

### Pattern 4: バランス優先

```
各部を均等に進める
適用: 全体像を常に把握したい場合
```

### Pattern 5: ハイブリッド

```
状況に応じて、上記を組み合わせる
適用: 柔軟に対応したい場合（推奨）
```

## Common Scenarios

### Scenario 1: 順調に進んでいる

**推奨**: 勢いを維持
- 現在の章の続きを執筆
- 難易度が同程度のセクション選択

### Scenario 2: 行き詰まっている

**推奨**: 気分転換
- 全く異なるトピックへ
- 軽いタスク（付録など）
- レビュー・修正作業

### Scenario 3: 特定の部が偏って進んでいる

**推奨**: バランス調整
- 遅れている部のセクション着手
- ただし、無理は禁物

### Scenario 4: 時間が限られている

**推奨**: 短時間で完了するタスク
- 既存セクションの修正
- 軽いセクション（コラムなど）
- レビュー作業

### Scenario 5: まとまった時間がある

**推奨**: 重要なセクション着手
- 核となる章の執筆
- 複数セクションの連続執筆
- 全体の見直し

## Tips

### 迷ったら

1. **直感を信じる**: 書きたいものを書く
2. **小さく始める**: 完璧を求めない
3. **柔軟に変更**: 計画は変えてOK

### 計画の頻度

- **毎日**: 今日のタスク確認
- **週1回**: 週次計画と振り返り
- **月1回**: 全体の進捗確認と調整

### モチベーション維持

- 完成の達成感を大切に
- 小さな成功を祝う
- 自分を追い込みすぎない
