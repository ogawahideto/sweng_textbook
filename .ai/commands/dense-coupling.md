# /dense-coupling — 書籍らしい密結合への変換

セクション間の繋がりを強化し、Webコンテンツ的な疎結合から書籍らしい密結合へ書き直します。

## 使い方

```
/dense-coupling [ファイルパス または 章ディレクトリパス]
```

引数がない場合は現在作業中の章を対象にします。

## 適用する3つの変換ルール

| # | 対象パターン | 処理内容 |
|---|------------|---------|
| 1 | `## 導入: タイトル` | 見出し行を削除（本文はそのまま） |
| 2 | `## 理論的背景: XXX` / `## 理論的背景` | プレフィックス削除 または 見出し行削除 |
| 3 | `## まとめ` の箇条書き | 流れる散文段落（2〜3段落）に書き直し + 次節への橋渡し追加 |

## 変換例（ルール3）

**変換前:**
```markdown
## まとめ

1. **アクティブ・リスニングは技術**: 共感・言い換えなど、具体的なテクニックで実践できる。
2. **5W1Hで深掘りする**: 特に「Why」は強力。
3. **感情にフォーカス**: 機能ではなく、ユーザーの感情を聴く。

次節では、AIにペルソナを演じさせて対話する技法を学びます。
```

**変換後:**
```markdown
## まとめ

「聴く」とは、受動的に音を受け取ることではありません。共感・傾聴・言い換えという具体的な
テクニックを組み合わせ、相手の言葉の背後にある感情や意図へと能動的に踏み込む技術です。
特に「なぜ？」という問いかけは強力で、表層の機能要求を突き破り、ユーザーが本当に体験
したいことへと道を切り開きます。

技法を身につけた次の問いは「では、誰に聴くのか？」です。1.3節では、AIにペルソナを
演じさせることで、何度でも仮想のユーザーと対話する技術を学びます。
```

## 実行ステップ

1. 対象ファイルを読む
2. 3つのパターンを検索・確認
3. 変換を適用（コードブロック・図・リソースセクションは変更しない）
4. 変換レポートを表示
5. ユーザー確認後に `/commit` でコミット

## ポジティブ・ライティング方針

`style-guide.md` に従い、散文化の際もポジティブな表現を維持:
- 「問題」→「機会・テーマ」
- 「難しい」→「奥深い・挑戦しがいがある」
- 「失敗する」→「成功に近づく」

詳細は `.claude/skills/dense-coupling/SKILL.md` を参照。
