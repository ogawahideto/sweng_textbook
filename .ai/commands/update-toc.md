# Update Table of Contents

本書の目次（toc.md）を最新の状態に更新します。

## 使い方

```
/update-toc
```

## 実行タイミング

以下のタイミングで自動的に実行することを推奨:

1. **新しいセクションを追加した後**
2. **セクションのタイトルを変更した後**
3. **章の構成を変更した後**
4. **セッション終了時**（進捗確認の一環として）

## 実行手順

### Step 1: 全章ファイルをスキャン

```
chapters/prologue/
chapters/part1/chapter01/
chapters/part1/chapter02/
...
chapters/part4/
```

各ディレクトリ内の `.md` ファイルを収集。

### Step 2: 各ファイルからタイトルを抽出

各 `.md` ファイルの1行目（`# タイトル`）を読み取る。

### Step 3: 目次を生成

以下の形式で `toc.md` を生成:

```markdown
# 目次

## プロローグ
- [プロローグ：ソフトウェアエンジニアリングは「魔法の地図」である](chapters/prologue/prologue.md)

## 第1部 構想の美学

### 第1章 ドメインという名の異世界探検——要求工学入門
- [第1章 概要](chapters/part1/chapter01/chapter01.md)
- [1.1 なぜ聴くのか？——要求工学という冒険の始まり](chapters/part1/chapter01/1-1.md)
- [1.2 どう聴くか？——アクティブ・リスニングの技法](chapters/part1/chapter01/1-2.md)
- [1.3 誰に聴くか？——AIペルソナとの対話](chapters/part1/chapter01/1-3.md)
- [1.4 どう整理するか？——ゴール指向要求分析](chapters/part1/chapter01/1-4.md)
- [1.5 どう可視化するか？——要求を描く古代文字（UML前編）](chapters/part1/chapter01/1-5.md)

### 第2章 ...
```

### Step 4: 執筆状態サマリーを追加

各セクションの執筆状態（🟨 AI草稿、🟩 人間レビュー済、✅ 完成）を集計:

```markdown
---

## 執筆進捗

| 状態 | セクション数 |
|------|-------------|
| ✅ 完成 | 1 |
| 🟩 人間レビュー済 | 0 |
| 🟨 AI草稿 | 6 |
| 📝 未着手 | -- |

**最終更新**: YYYY-MM-DD
```

### Step 5: ファイルに書き出し

`toc.md` をプロジェクトルートに保存。

## 出力形式

```markdown
# 『「楽しい」ソフトウェアエンジニアリングの教科書』目次

## プロローグ
- [タイトル](パス) — 執筆状態

## 第1部 構想の美学

### 第1章 章タイトル
- [セクションタイトル](パス) — 執筆状態

...

---

## 執筆進捗サマリー

| 状態 | 数 |
|------|---|
| ✅ 完成 | N |
| 🟨 AI草稿 | N |

**最終更新**: YYYY-MM-DD
```

## 注意事項

- ファイルが存在しない場合は目次から除外
- タイトルが取得できない場合はファイル名を使用
- 相対パスで記述（GitHubなどで閲覧可能にするため）
