# Commit（論理単位ごとのコミット）

CLAUDE.md のコミットルール「1コミット = 1つの論理的な変更単位」に従い、変更をグループ分けしてコミットします。

## 手順

### Step 1: 変更の全体像を把握

以下のコマンドで現在の変更内容を確認する:

```bash
git status
git diff
git diff --staged
git log --oneline -5
```

- 未追跡ファイル、ステージ済み変更、未ステージ変更をすべて把握する
- 直近のコミットメッセージのスタイルを確認する

### Step 2: 変更をグループ分け

変更ファイルを「論理的な変更単位」ごとにグループ化する。分類基準:

| グループ | 対象ファイル例 |
|---------|---------------|
| 章・セクションの執筆 | `chapters/chX/X.Y.md`（同じ章・節ごと） |
| 進捗管理 | `progress.md`, `toc.md`, `plan.md` |
| セッションログ | `.ai/sessions/*.md` |
| スタイルガイド・設定 | `style-guide.md`, `CLAUDE.md`, `GEMINI.md`, `.ai/**` |
| サンプルコード | `sample-code/**` |
| 図・画像 | `chapters/chX/images/**` |

**注意:**
- 1つのファイルが複数グループに属する場合は、最も関連の強いグループに入れる
- 同じ章でも異なるセクションの変更が独立している場合は、さらに分けてもよい
- 変更が1グループだけなら、グループ分けの提示は省略してそのままコミットする

### Step 3: ユーザーに確認

グループ分けの結果をテーブル形式で提示し、ユーザーの承認を得る。

```
| # | グループ | ファイル | コミットメッセージ案 |
|---|---------|---------|-------------------|
| 1 | 3.1節の執筆 | chapters/ch3/3.1.md | docs(ch3): 3.1節にソートアルゴリズムの説明を追加 |
| 2 | 進捗管理の更新 | progress.md, toc.md | docs: update progress and TOC |
```

ユーザーが修正を求めた場合はグループ分けやメッセージを調整する。

### Step 4: 順番にコミット

承認後、各グループを順番にコミットする:

1. グループに含まれるファイルを `git add`（ファイル名を個別指定）
2. コミットメッセージは HEREDOC で渡す
3. Co-Authored-By を付与する

```bash
git add [files...]
git commit -m "$(cat <<'EOF'
docs(ch3): 3.1節にソートアルゴリズムの説明を追加

Co-Authored-By: Claude Opus 4.6 <noreply@anthropic.com>
EOF
)"
```

**コミットメッセージのスタイル:**
- プレフィックス: `docs:`, `docs(chX):`, `feat:`, `fix:`, `chore:` 等
- 日本語または英語（直近のコミットスタイルに合わせる）
- 1行目は簡潔に（70文字以内推奨）
- 必要に応じて空行の後に詳細を記述

### Step 5: 結果を報告

全コミットの一覧を `git log --oneline` で表示する。
