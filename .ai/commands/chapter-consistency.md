# Chapter Consistency Check

章内の複数セクション間の整合性を確認します。

## 使い方

```
/chapter-consistency [章番号]
/chapter-consistency 1      # 第1章を確認
/chapter-consistency 2      # 第2章を確認
```

## 実行手順

### Step 1: 対象ファイルの収集

指定された章のすべてのセクションファイルを読み込む:

```
chapters/part1/chapter0X/chapter0X.md  (章の冒頭ページ)
chapters/part1/chapter0X/X-1.md
chapters/part1/chapter0X/X-2.md
...
```

### Step 2: 整合性チェック項目

以下の観点で各ファイルを確認:

#### 2.1 相互参照の正確性

- 「X.Y節で」「前節で」「次節で」などの参照が正しいか
- 参照先に該当する内容が実際に存在するか
- 例: 「1.1節の対話で」→ 実際に1.1節に対話例があるか確認

#### 2.2 用語の一貫性

| 確認項目 | チェック内容 |
|---------|------------|
| QuestForge関連 | クエスト、勇者、経験値、レベルアップなどの表記揺れ |
| ペルソナ名 | Aさん、翔太、美咲、健一などの名前と属性の一致 |
| 技術用語 | UML、ユースケース、アクティビティなどの表記 |
| 英語表記 | Requirements Engineering、Active Listeningなどの統一 |

#### 2.3 メタ情報の整合性

- **執筆状態**: 全セクションの状態を一覧化
- **最終更新日**: 古い日付がないか確認
- **章タイトル**: 冒頭ページと各セクションの整合

#### 2.4 ポジティブ・ライティング

style-guide.mdの方針に沿っているか:
- ネガティブワード（危険、失敗、罠、落とし穴、悪い例など）がないか
- 発見した場合は代替表現を提案

#### 2.5 構成の一貫性

各セクションが以下の構造を持っているか:
- 導入
- 理論的背景
- 実践例（QuestForge関連）
- AI時代のアプローチ / ハンズオン
- まとめ
- さらに学ぶためのリソース
- AIへの詠唱例

#### 2.6 参考文献の重複・欠落

- 同じ書籍が複数セクションで異なる記述になっていないか
- 章全体で参照すべき重要文献が漏れていないか

### Step 3: 結果をフォーマット

```markdown

**対象**: 第X章 [章タイトル]
**確認日**: YYYY-MM-DD

```

### Step 4: 修正の適用

ユーザーに結果を提示し、修正が必要な箇所があれば確認の上で適用。

## チェックリスト用クイックコマンド

章全体を素早く確認するためのgrepパターン:

```bash
# 相互参照の抽出
grep -n "節で\|前節\|次節\|第.*章" chapters/part1/chapter0X/*.md

# ネガティブワードの検出
grep -n "危険\|失敗\|罠\|落とし穴\|悪い例\|エラー\|警告\|罪悪感\|不便\|苦手" chapters/part1/chapter0X/*.md

# メタ情報の抽出
grep -n "執筆状態\|最終更新" chapters/part1/chapter0X/*.md
```

## 注意事項

- 大量の修正が必要な場合は、優先度をつけて段階的に対応
- 用語の「意図的な使い分け」がある場合は、その理由を確認してから修正
- ポジティブ・ライティングは文脈を考慮（ユーザーの悩みを描写する場面などは許容）
