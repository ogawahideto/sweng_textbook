# Progress Check

執筆の進捗状況と次のタスクを表示します。

## 実行内容

### Step 1: progress.md を読み込む

`progress.md` から現在の進捗状況を取得。

### Step 2: 進捗サマリーを表示

```markdown
## 執筆進捗サマリー

### 全体進捗
| 状態 | セクション数 |
|------|------------|
| ✅ 完成 | X |
| 🟨 AI草稿 | Y |
| 🟦 修正中 | Z |
| ⬜ 未着手 | W |

**進捗率**: XX% (完成 + 草稿)
```

### Step 3: 現在のフォーカスエリアを表示

現在作業中の章・節を特定:

```markdown
### 現在の作業エリア

**第X章: [章タイトル]**

| 節 | タイトル | 状態 |
|----|---------|------|
| X.1 | ... | ✅ |
| X.2 | ... | 🟨 |
| X.3 | ... | ⬜ ← 次 |
```

### Step 4: 次のタスクを提案

`progress.md` の「次のタスク」セクションから未完了タスクを抽出:

```markdown
### 次のタスク（推奨順）

1. **[最優先タスク]**
   - 理由: [なぜこれが次か]
   - 実行: `/draft X.Y` または手動で開始

2. **[次点タスク]**
   - 理由: ...
```

### Step 5: 最近の活動を表示

Gitログから最近のコミットを表示:

```bash
git log --oneline -5
```

```markdown
### 最近の活動（直近5コミット）
- [コミット1]
- [コミット2]
- ...
```

## オプション

### `/progress --detail`

各セクションの詳細状態を表示（ファイルサイズ、最終更新日など）

### `/progress --chapter X`

特定の章のみの進捗を表示
