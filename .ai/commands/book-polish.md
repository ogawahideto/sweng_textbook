# /book-polish コマンド

書籍的可読性の7ルール（style-guide.md §書籍的可読性のルール）に基づいて、指定したMarkdownファイルまたはディレクトリをスキャンし、違反をレポートします。セクション末尾の順序（ルール⑥）は自動修正します。

## 使い方

```
/book-polish [パスまたはディレクトリ]
```

## スキャンルール一覧

| ルール | チェック内容 | 対応 |
|--------|------------|------|
| ① 見出し密度 | H2セクション内のH3が5個超 | 警告+格下げ候補の提示 |
| ② 太字の過多 | 1,000字あたり `**` が15個超 | 警告+削減候補の提示 |
| ⑤ 図の前後文 | `![` の直前行が空行のみ | 警告+誘導文の追加提案 |
| ⑥ セクション末尾の順序 | `さらに学ぶ` が `AI詠唱例` より前 | **自動修正** |
| ⑦ コラムの上限 | `> ` が15行以上連続 | 警告+分割/移動の提案 |
| ⑧ 前節参照 | `^前節で` `^ここまでで` などのパターン | 警告+書き換え案の提示 |

## 実行手順

1. 引数に指定されたパスの `.md` ファイルを列挙する
2. 各ファイルに対してルール①②⑤⑥⑦⑧をチェックする
3. ルール⑥の違反があれば自動修正する
4. 結果をレポート形式で出力する

### ルール①: 見出し密度

H2区切りでセクションを分割し、各セクション内のH3（`### `）の数を数える。5個を超えるセクションを報告し、連番・手順型のH3を太字化する候補として提示する。

### ルール②: 太字の過多

全テキストから `**` 開きタグの数と文字数を計算。1,000字あたり15個超で警告。太字を減らす候補として「2回目以降の専門用語」「リスト冒頭の太字ラベル」を提示する。

### ルール⑤: 図の前後文

`![` で始まる行を探し、直前行が空行・区切り線・見出しのみの場合は誘導文なしと判定。直後行が空行・次の見出し・リストの場合は読み解き段落なしと判定。

### ルール⑥: セクション末尾の順序（自動修正）

ファイル内で `## AIへの詠唱例` と `## さらに学ぶためのリソース` の両方が存在し、後者が前者より前にある場合、ブロックを入れ替えて上書き保存する。

### ルール⑦: コラムの上限

`> ` で始まる行が15行以上連続する箇所を報告。開始行番号と行数を表示し、章末付録への移動または本文への吸収を提案する。

### ルール⑧: 前節参照

以下のパターンにマッチする行を報告し、問いかけ形式への書き換え案を提示する：
- `前節で[はにもが]`
- `ここまで[でに]`
- `本節では`
- `この節では`

## レポート形式

```
=== /book-polish レポート ===

📄 ファイル名
  ① 〜 （詳細）
  ② 〜 （詳細）
  ⑥ 自動修正しました ✅

合計: Nファイルをスキャン、M件の違反、K件を自動修正
```
