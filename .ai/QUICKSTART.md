# Quick Start Guide

AIと協働で書籍を執筆するためのクイックスタートガイドです。

## 初めて使う場合

### 1. プロジェクトを理解する

```bash
# まず、これらのファイルを読む
cat README.md              # 全体の使い方
cat plan.md                # 書籍の企画
cat .ai/context/project.md # プロジェクト概要
```

### 2. AIエージェントを確認

```bash
cat .ai/agents.md          # AIの役割・能力を確認
```

### 3. 最初のタスクを選ぶ

**オプションA: AIに提案してもらう**

AIに以下のように指示：

```
@.ai/workflows/plan-next.md のワークフローに従って、
次に取り組むべきタスクを提案してください。

現在の状況:
@progress.md
```

**オプションB: 自分で決める**

`progress.md` を見て、未着手（⬜）のセクションから選ぶ。

## 基本的な使い方

### パターン1: 新しいセクションを執筆

```
Step 1: AIに草稿生成を依頼

@.ai/workflows/draft.md のワークフローに従って、
第1章1.1節の草稿を生成してください。

@plan.md から該当セクションの情報を参照してください。

Step 2: 生成された草稿を保存

chapters/part1/chapter01/1-1.md に保存

Step 3: progress.md を更新

状態を 🟨 AI草稿 に変更

Step 4: 人間が修正・追記

草稿を読み、自分の視点や経験を追加

Step 5: progress.md を更新

状態を 🟦 人間修正中 に変更
```

### パターン2: セクションをレビュー

```
Step 1: AIにレビューを依頼

@.ai/workflows/review.md のワークフローに従って、
以下のセクションをレビューしてください。

セクション: 1.1 隠れた願いを翻訳する

[ここにセクションの全文を貼り付け]

Step 2: フィードバックを検討

AIの提案を読み、採用するものを選択

Step 3: 改善を適用

セクションを修正

Step 4: progress.md を更新

状態を 🟩 レビュー待ち または ✅ 完成 に変更
```

### パターン3: 次のタスクを決める

```
@.ai/workflows/plan-next.md のワークフローに従って、
次に取り組むべきタスクを提案してください。

@progress.md
```

## AIへの指示の仕方

### 基本形式

```
@[ファイルパス] のワークフロー/ガイドラインに従って、
[具体的な依頼内容]。

[必要なコンテキスト情報]
```

### 使用する @ 記法

- `@.ai/agents.md` - AIの役割と振る舞い
- `@.ai/workflows/draft.md` - 草稿生成ワークフロー
- `@.ai/workflows/review.md` - レビューワークフロー
- `@.ai/workflows/plan-next.md` - タスク計画ワークフロー
- `@plan.md` - 書籍企画
- `@style-guide.md` - 執筆スタイル
- `@progress.md` - 進捗状況
- `@templates/section-template.md` - セクションテンプレート

### 例1: 草稿生成

```
@.ai/workflows/draft.md のワークフローに従って、
3.1節「詠唱（プロンプト）による具現化」の草稿を生成してください。

セクション目的:
@plan.md より「伝統的なアルゴリズムを理解した上で、
AIに最適な実装を生成させる知的指揮権を獲得する」

スタイル要件:
@style-guide.md に準拠

テンプレート:
@templates/section-template.md の構造に従う
```

### 例2: レビュー

```
@.ai/workflows/review.md のワークフローに従って、
以下のセクションを包括的にレビューしてください。

@.ai/agents.md の "When Reviewing Content" ガイドラインを参照

セクション: 3.1 詠唱（プロンプト）による具現化

[セクション全文]

特に以下の観点を重視:
- 技術的正確性
- @style-guide.md への準拠
```

### 例3: タスク計画

```
@.ai/workflows/plan-next.md のワークフローに従って、
次のタスクを提案してください。

現在の状況:
@progress.md

考慮事項:
- 今は理論よりも実践的な内容を書きたい気分
- まとまった時間（3-4時間）が取れる
```

## 実践例：最初のセクションを書く

### ステップ1: タスク決定

**あなた**:

```
@.ai/workflows/plan-next.md に従って、
最初に取り組むべきタスクを提案してください。

@progress.md
```

**AI**:

```
推奨タスク: プロローグの執筆
理由: 書籍の方向性を確立...
[詳細な提案]
```

**あなた**: 「わかりました。プロローグを書きます」

### ステップ2: 草稿生成

**あなた**:

```
@.ai/workflows/draft.md に従って、
プロローグの草稿を生成してください。

目的: @plan.md のプロローグセクションを参照
スタイル: @style-guide.md に準拠
構造: @templates/section-template.md
```

**AI**:

```
# プロローグ：ソフトウェアエンジニアリングは「魔法の地図」である

[2500文字の草稿]

---
Meta Information:
- 文字数: 2500
- 主要概念: ...
```

**あなた**: 草稿を `chapters/prologue/prologue.md` に保存

### ステップ3: 進捗更新

`progress.md` を編集：

```markdown
| セクション | 状態 | 担当 | 備考 |
|-----------|------|------|------|
| プロローグ全体 | 🟨 | AI+人間 | 草稿完成 2026-01-18 |
```

### ステップ4: 人間が修正

`chapters/prologue/prologue.md` を開いて：

- 自分の経験を追加
- 具体例を差し替え
- 文体を調整
- コードを検証

### ステップ5: 進捗更新

```markdown
| プロローグ全体 | 🟦 | AI+人間 | 人間修正中 |
```

### ステップ6: レビュー依頼

**あなた**:

```
@.ai/workflows/review.md に従って、
プロローグをレビューしてください。

[修正後の全文を貼り付け]
```

**AI**:

```
## Strengths
1. 魅力的な導入...

## Improvements Needed
### High Priority
1. 技術用語の説明が...
[詳細なフィードバック]
```

### ステップ7: 最終修正

フィードバックを元に再修正

### ステップ8: 完成

`progress.md` を更新：

```markdown
| プロローグ全体 | ✅ | AI+人間 | 完成 2026-01-18 |
```

## Tips

### 効率的な協働

1. **コンテキストを明示**: 必要なファイルを @ で参照
2. **ワークフローに従う**: `.ai/workflows/` を活用
3. **小さく反復**: 一度に全てを完璧にしない
4. **人間が最終判断**: AIは提案、決定は人間

### トラブルシューティング

**AIの出力が期待と違う**
→ より具体的な指示、例を示す

**スタイルが合わない**
→ `@style-guide.md` を明示的に参照させる

**技術的に不正確**
→ 具体的な修正点を指示、または自分で修正

**進捗管理が面倒**
→ AIに `progress.md` 更新を依頼

### モチベーション維持

- 小さな達成を祝う
- 楽しいセクションから書く
- 行き詰まったら別のタスクへ
- 完璧を求めすぎない
