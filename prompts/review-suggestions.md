# レビュー・改善提案プロンプト

このプロンプトを使って、執筆したセクションのレビューと改善提案を得ます。

## 基本プロンプト

```
あなたは『「楽しい」ソフトウェアエンジニアリング —— デジタル・アルケミー：伝統の美学とAIで紡ぐ創造の地図』という攻略本の編集者です。

以下のセクションをレビューし、改善提案を行ってください。

【レビュー対象】
[章番号].[節番号] [タイトル]

【セクション内容】
[ここに執筆済みのセクションをペースト]

【レビューの観点】
1. **構成・論理展開**
   - 導入は読者を引き込むか
   - 論理の流れは自然か
   - まとめは適切か

2. **内容の質**
   - 技術的な正確性
   - 説明の分かりやすさ
   - 実践例の実用性
   - AI時代の視点が適切に盛り込まれているか

3. **文体・トーン**
   - 親しみやすさと専門性のバランス
   - 比喩や具体例の効果
   - 読者への語りかけの自然さ

4. **ボリューム**
   - 文字数は適切か（2000-4000文字目安）
   - 情報量は過不足ないか

5. **全体との整合性**
   - 本書のコンセプトと合致しているか
   - 前後のセクションとの接続は良好か

【フィードバック形式】
- 良い点（具体的に）
- 改善すべき点（具体的な修正案と共に）
- 追加すべき要素
- 削除または縮小すべき要素
- その他の提案

【期待する出力】
客観的で建設的なフィードバックを、具体的な修正案と共に提示してください。
```

## 使い方

1. `[章番号].[節番号] [タイトル]` を該当するセクションに置き換える
2. `[ここに執筆済みのセクションをペースト]` に実際のセクション内容をペースト
3. AIに上記のプロンプトを入力
4. フィードバックを元にセクションを修正
5. progress.md の状態を 🟦 人間修正中 に更新

## レビュー後のワークフロー

1. AIの提案を検討
2. 採用する改善点を選択
3. 自分の視点で追加の修正を加える
4. 修正版を保存
5. 必要に応じて再度レビューを依頼
6. 満足したら progress.md の状態を 🟩 レビュー待ち に更新
