# 次のタスク提案プロンプト

このプロンプトを使って、執筆の次のステップを決定します。

## 基本プロンプト

```
あなたは『「楽しい」ソフトウェアエンジニアリング —— デジタル・アルケミー：伝統の美学とAIで紡ぐ創造の地図』という攻略本の執筆プロジェクトマネージャーです。

現在の執筆状況を分析し、次に取り組むべきタスクを提案してください。

【現在の状況】
[progress.md の内容をペースト]

【最近完了したタスク】
[最近完了したセクションや作業を記載]

【考慮すべき要素】
1. 執筆の優先順位
   - 論理的な流れ（前の章が完成してから次へ）
   - 独立性（並行して進められるセクション）
   - 難易度（簡単なものから着手するか、難しいものを先にするか）

2. 執筆のバランス
   - 各部のバランス
   - 理論と実践のバランス
   - 完成度のバランス（一部を完璧にするか、全体を進めるか）

3. 執筆者のモチベーション
   - 今、興味が高いトピック
   - 執筆しやすいセクション
   - チャレンジングなセクション

【期待する出力】
以下の形式で提案してください：

1. **推奨する次のタスク**（最も優先すべきもの）
   - タスク名:
   - 理由:
   - 期待される成果物:
   - 推定作業時間:

2. **代替案1**（並行して進められるタスク）
   - タスク名:
   - 理由:
   - 期待される成果物:

3. **代替案2**（後回しにできるタスク）
   - タスク名:
   - 理由:
   - 期待される成果物:

4. **全体のロードマップ**（今後3-5タスクの流れ）
   - ステップ1:
   - ステップ2:
   - ステップ3:
   - ...

5. **注意点・懸念事項**
   - [ある場合は記載]
```

## 使い方

1. progress.md の最新の状態をコピー
2. `[progress.md の内容をペースト]` に貼り付け
3. `[最近完了したセクションや作業を記載]` を更新
4. AIに上記のプロンプトを入力
5. 提案されたタスクを検討
6. 自分の判断で次のタスクを選択
7. progress.md の「次のタスク」セクションを更新

## 意思決定のヒント

- **論理的に進めたい場合**: 推奨タスクを採用
- **並行作業したい場合**: 代替案も同時に進める
- **今の気分で決めたい場合**: 自分が書きたいセクションを優先
- **行き詰まった場合**: このプロンプトで新しい視点を得る

## タスク選択後のワークフロー

1. 選択したタスクを明確化
2. section-draft.md プロンプトでAI草稿を生成
3. 人間が修正・追記
4. review-suggestions.md プロンプトでレビュー
5. 再修正
6. 完成したら次のタスクへ
